# Extend jsPsych

jsPsych is small at its core and powerful at its edges. Almost everything a participant sees or does in an experiment comes from a **plugin**, **extension**, or **timeline** — and any of them can be written by the community. This section is about that ecosystem: finding what others have built, building your own, sharing it, and contributing to the core project.

## The building blocks

| Building block | What it is | When you'd build one |
|---|---|---|
| **[Plugin](../learn/concepts/plugins.md)** | Defines a single kind of trial or event (showing a stimulus, collecting a response). | You need a type of trial that no existing plugin provides. |
| **[Extension](../learn/concepts/extensions.md)** | Adds a capability that runs *alongside* trials (e.g. eye tracking, mouse tracking). | You want to record or augment data across many trials, regardless of plugin. |
| **Timeline** | A reusable, parameterized procedure built from existing plugins. | You want to package and share a whole task or paradigm. |

Plugins and extensions you build can live in your own project, or be shared with the community through the [`jspsych-contrib`](https://github.com/jspsych/jspsych-contrib) repository. Reusable timelines are shared through [`jspsych-timelines`](https://github.com/jspsych/jspsych-timelines).
